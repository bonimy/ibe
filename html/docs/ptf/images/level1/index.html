Overview

Processed data corresponding to the original single CCD exposures. These individual image files are sometimes simply referred to as "single exposure", "processed images", or even sometimes "frameproc." Level-1 data can consist of calibrated FITS images, mask images, and calibrated photometric catalogs in binary FITS table format. At this time, all publicly available data are Level 1 data, but not all Level 1 data are publicly available. If you have proprietary access to these data, you must log in to be able to download these data. A complete listing of available columns can be accessed by clicking the link above.

Metadata queries can be issued at the following URL:

<server><root>/search/ptf/images/level1?<parameters>

For example, <root>/search/ptf/images/level1?POS=0,0 returns metadata for images containing (RA,Dec) = (0, 0). Query parameters are documented here.

Generating URLs for data products

Intensity FITS images

These are located at:

<server><root>/data/ptf/images/level1/{pfilename:s}

where pfilename is the values of the column pfilename.

Examples:

  • <root>/data/ptf/images/level1/proc/2012/09/20/f1/c0/p5/v1/PTF_201209202356_i_p_scie_t053919_u014094242_f01_p022592_c00.fits

By appending pfilename and size parameters one can also download cutouts (see the detailed documentation).

  • <root>proc/2012/09/20/f1/c0/p5/v1/PTF_201209202356_i_p_scie_t053919_u014094242_f01_p022592_c00.fits?center=0,0&size=200pix

Mask FITS images

These are located at:

<server><root>/data/ptf/images/level1/{afilename1:s}

where afilename1 is the values of the column afilename1.

Examples:

  • <root>/data/ptf/images/level1/proc/2012/09/20/f1/c0/p5/v1/PTF_201209202356_i_p_mask_t053919_u014094242_f01_p022592_c00.fits

By appending afilename1 and size parameters one can also download cutouts (see the detailed documentation).

  • <root>proc/2012/09/20/f1/c0/p5/v1/PTF_201209202356_i_p_mask_t053919_u014094242_f01_p022592_c00.fits?center=0,0&size=200pix

    • Append center and size parameters to these URLs to download a cutout instead of a full image (see the cutouts documentation).

SExtractor Catalog

These are located at:

<server><root>/data/ptf/images/level1/{afilename3:s}

where afilename3 is the values of the column afilename3.

Examples:

  • <root>/data/ptf/images/level1/proc/2012/09/20/f1/c0/p5/v1/PTF_201209202356_c_p_scie_t053919_u014094242_f01_p022592_c00.ctlg
